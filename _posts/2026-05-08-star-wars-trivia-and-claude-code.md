## Why this post

This is the first Fertile Ground trivia system I built — the Star Wars night — and I'm taking ownership of the event going forward. The system ended up as two small repos that talk to each other only through a host's hands and a paper answersheet:

- **`star-wars-trivia-game`** — the live presentation deck (React + Vite, custom `<deck-stage>` web component, BroadcastChannel sync between a display window and a control window, picture round, tiebreakers).
- **`trivia-scorer`** — the admin UI for entering paper-graded scores and a public leaderboard (React + TypeScript on Cloudflare Pages, D1 for storage, edge-cached `/api/tournaments/active/leaderboard` polled every 5s, QR code on the leaderboard so the brewery's big screen is scannable).

## Prompts

### 1. Origin story — what's "Fertile Ground" and how did I end up running trivia?

Fertile Ground Beer Co. will celebrate 4 years of operation on May 15th & 16th — Jackson's only brewery is now the latest venue for which I've written a trivia game. I have been working part-time in the taproom for over a year now and I'm ready to bring more of my talents to the 3 80" screens mounted on the walls. 

May the 4th — also known as Star Wars Day — was the perfect time to trial run a new trivia hosting crew. Previously, friends of the brewery volunteered to run trivia games on an every-other-week cadence, and eventually life caught up with their efforts. I'm reintroducing trivia events with my colleague Jack Smith, and I can't promise a regular cadence but I do want to fit in at least 2 each month.

### 2. The brief — what does a trivia night actually need?

The constraints I was building for, bulleted:

- Five rounds of ten questions plus a picture round and tiebreakers
- A projector showing the question to the room while I drive from a laptop
- Paper grading by hosts (no audience-response gear, no logins for players)
- A public leaderboard that updates between rounds without me babysitting it
- Has to survive a brewery Wi-Fi outage mid-event

My initial goal was simply a themed trivia-hosting app with placeholders for questions I would write by hand later -- I wanted to use AI to make the tool I needed, not to do the fun part for me. Writing the questions sometimes starts with me deciding on an answer I want to test the general public for knowledge -- does everyone know the difference between Anakin Skywalker and Darth Vader well enough to differentiate between their quotes or catchphrases? I created a themed presentation and controller so that it was easy to keep the game on track. I included a timer so that things stayed moving, with the timer appearing on each slide with a trivia question so that there's a reasonable expectation that we can move on and everyone has had a fair chance to read the question.

The `star-wars-trivia-game` project generated by Claude Code is local-only and version controlled with git. It will not appear on my public GitHub profile but I may upload the `trivia-scaffold` repo that came after — it will be part of the DNA of trivia games moving forward, serving as a useful app for generic trivia as well as customizable with a skill to match a particular theme.

`trivia-scorer` was created with the simple goal of tracking scores for multiple teams across a variable number of rounds. The setup mode allows to update the number of questions/rounds ahead of starting a game. There is also a leaderboard mode for the displays in the taproom.

### 3. Why split into two apps?

I decoupled the scoring app from the trivia deck because I assumed those two functions were strongly coupled to a single event but ultimately could be performed or controlled by more than one person -- I learned the most from hosting the event and controlling both apps and now I've learned what I think I needed to know about the trivia scoring for our games: if we just assign one person to enter the data per round, it will probably mean entering 15-25 scores per round. That's not exactly complicated, though it should be straightforward if it is to be a low-friction effort for a live gaming event.

The fact also remains that the trivia questions are relatively secret and the leaderboard should not be — so now the trivia scoring app and leaderboard are both deployed to a Cloudflare Pages instance with access controls enforced by Cloudflare Access. This is great for free tier accounts that only need a handful of seats for access to an app.

### 4. The deck — `star-wars-trivia-game`

- I didn't specifically ask for a custom `<deck-stage>` web component handling slide layout for scaling and print geometry — so why I reached for vanilla JS instead of a React component is more up to Claude than me
- Two windows, one keyboard: the display URL is `/`, the control URL is `/#/control`, and they talk over `BroadcastChannel` (`rounds:update`, `nav:next/prev/goto`, `timer:toggle/reset/adjust`, `pictures:update`) — this was a design I asked for to give myself the most flexibility as far as control over a public visual experience
- Picture round: I wanted setting up the round to be as simple as pasting an image, drag-to-pan to crop, and export a 1920×1080 handout PNG that prints cleanly
- Tiebreakers: three questions plus a Final Wager (similar to Final Jeopardy)
- Question editor in the control window that buffers edits to localStorage with a dirty flag, so I never accidentally publish a half-typed change to the projector mid-round
- Export questions: the trivia-scorer scaffold is reusable, so questions can be written and exported for games that happen in the future; the scaffold plus the `/new-trivia-game` skill allow for custom theming based on specific books/movies/tv shows/etc

### 5. The scorer — `trivia-scorer`

- Cloudflare Pages Functions + D1, with the entire `Event` blob stored as JSON in a single row (Phase 1 simplicity over a normalized schema)
- Reducer-driven state machine: `setup → scoring → final → tiebreaker (if needed)`
- Public leaderboard at `/leaderboard` polling every 5s, with the edge cache set to 5s so the polling loop doesn't hammer D1
- QR code on the leaderboard so anyone in the brewery can pull the standings up on their phone
- Cloudflare Access as an identity provider
- CSV export plus an `npm run import:csv` round-trip for archived tournaments from before the D1 migration
